# AGENTS.md — Build orientation for Cobolens> This file tells an AI coding agent how to build **Cobolens**. Read it first, then read `PRD.md`. (Works equally as `CLAUDE.md`.)## What you are buildingCobolens: a **free, open-source, local desktop app** that points at a COBOL codebase and produces an interactive dependency map, diagrams, and section summaries, with grounded AI chat using a bring-your-own model (Anthropic / OpenAI / OpenRouter / local Ollama). It can run **fully local / air-gapped** — code never leaves the machine. It is an *understanding* tool, not a migration or code-generation tool.## The single most important rule**Principle 0 — Lightweight, not over-engineered (overrides everything).** Smallest thing that delivers the end-to-end experience. Prefer boring, proven, minimal solutions. Lean on existing libraries; do not build your own. No backend server, no external database, no plugin system, no microservices, no speculative abstraction or "future-proofing" unless a stated requirement forces it. Every dependency, layer, and feature must earn its place. When in doubt, leave it out.## Source of truth & file map- **`PRD.md`** — the canonical specification. It governs. Everything you build must trace to it.- **`docs/`** — reference research only (incumbent teardown, parser landscape, UX study, desktop/model plumbing, test repos, synthesis). Use for context and rationale; it does not override the PRD.- If `PRD.md` and `docs/` ever conflict, **the PRD wins.***(Repo setup: place the PRD at the repo root as `PRD.md`, and the `00`–`05` research markdown files in `docs/`.)*## How to work — the build loopBuild the milestones in **`PRD.md` §19 in order (M0 ? M5;** M6 is later). For each milestone:1. **Read** the relevant PRD sections (the milestone references them).2. **Implement** the smallest version that satisfies the milestone, honoring Principle 0.3. **Verify against the milestone's Acceptance Criterion (AC).** The ACs are the done-check — most run against the COBOL Legacy Benchmark Suite (see PRD §14). Do not advance until the AC passes.4. **Commit** with a message naming the milestone (e.g. `M2: graph view focus-and-expand`).5. **Move to the next milestone.** Do not skip ahead or build later milestones' features early.## Guardrails — honor these from the PRD- **Architecture is a thin assembly** of off-the-shelf parts glued by one JSON contract (PRD §10, §20). Don't add layers.- **Parser is a swappable sidecar** behind the contract in §20. v1 uses the lightweight tree-sitter path (§11, path 3). Never let UI code depend on parser internals.- **The graph is ground truth.** All diagrams/lineage/impact come from the parsed graph (PRD §6, §9, §22). The AI explains; it must never invent nodes, edges, or structure. Every AI claim cites file + line.- **Privacy is real.** In local mode, make zero outbound network calls for inference or embeddings, and keep that verifiable (PRD §8.6, §24). API keys live in the OS keychain, never in config/logs.- **Stack is decided** (PRD §13): Tauri v2 + React/TS + Sigma.js/graphology + Vercel AI SDK + Ollama. Don't substitute without a stated reason. Don't add dependencies beyond PRD §18 without justification.- **Graceful degradation:** unparsed files are listed, not fatal; missing copybooks/programs become external nodes; the app is fully usable for the map/docs even with no model configured (PRD §25).## Stop and ask the human when- You reach the **parser bake-off** decision (PRD §11 / §14) — confirm the tree-sitter COBOL grammar choice on real code before committing deeply to it.- You'd need to **add a dependency, a server, or a database** not in the PRD.- An AC **cannot be met** as specified, or the PRD is ambiguous on something load-bearing.- You're tempted to build anything in the **out-of-scope** list (PRD §7.2): write/generate COBOL, translation, verification, live mainframe connectivity, team/cloud features.## Definition of done (v1)A developer with no COBOL background can open Cobolens on the benchmark suite, see the dependency map, read section summaries, and ask "where does X happen and what depends on it?" — getting a correct, cited answer — all running locally if they choose.
